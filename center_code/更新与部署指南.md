# 项目更新与部署指南

本文档记录：拉取更新 → 本地跑通 → 上传云服务器的完整步骤与执行位置。

---

## 一、拉取远程更新

**在哪里执行：** 在项目根目录（即 `D:\project\333`）打开终端（PowerShell 或 CMD）。

| 步骤 | 指令 | 说明 |
|------|------|------|
| 1 | `git status` | 查看当前是否有未提交修改（可选） |
| 2 | `git stash` | 若有本地修改想暂时保留，先暂存（可选） |
| 3 | `git pull` | 拉取远程最新代码（默认拉当前分支） |
| 4 | `git stash pop` | 若执行了 stash，拉取后再恢复（可选） |

**若使用指定分支（例如 main）：**
```powershell
git checkout main
git pull origin main
```

## 二、本地跑通

项目包含：**后端（Flask）** + **前端（Vue + Vite）**。前端构建产物输出到 `center_code/backend/static`，后端会直接提供该静态目录。

### 2.1 后端依赖与运行

**在哪里执行：** `D:\project\333\center_code\backend`

| 步骤 | 指令 | 说明 |
|------|------|------|
| 1 | `python -m venv venv` | 若无虚拟环境，先创建（可选） |
| 2 | `.\venv\Scripts\activate` | 激活虚拟环境（PowerShell） |
| 3 | `pip install -r requirements.txt` | 安装/更新 Python 依赖 |
| 4 | 确认 `.env` 存在且配置正确 | 数据库、端口等（见下方说明） |
| 5 | `python app.py` | 启动后端（默认端口 8083，以 .env 为准） |

**说明：**
- 端口在 `.env` 中配置：`SERVER_PORT=8083`、`PORT=8083`。
- 数据库需为 MySQL，并已创建 `autovideo` 库；配置在 `.env` 的 `DB_*`。
- 若需修改 FFmpeg 等路径，在 `.env` 中修改 `FFMPEG_PATH`。

### 2.2 前端依赖与构建（若代码有前端更新）

**在哪里执行：** `D:\project\333\center_code\frontend`

| 步骤 | 指令 | 说明 |
|------|------|------|
| 1 | `npm install` | 安装/更新前端依赖 |
| 2 | `npm run build` | 构建生产包，输出到 `../backend/static` |

**本地开发时（可选）：**  
- 前端开发：在 `frontend` 目录执行 `npm run dev`，并确保前端请求的后端端口与 `.env` 中一致（如 8083）。  
- 后端单独运行：在 `backend` 目录执行 `python app.py`，访问 `http://localhost:8083` 会使用已构建的 `backend/static`。

### 2.3 验证本地跑通

1. 在 `backend` 目录已执行 `python app.py`。
2. 浏览器访问：`http://localhost:8083`（或你在 `.env` 里配置的端口）。
3. 能打开登录页、接口正常即表示跑通。

## 三、上传更新到云服务器

以下为通用流程，实际以你的服务器环境为准（如 Linux + Nginx + systemd / supervisor）。

### 3.1 方式 A：Git 拉取（服务器上已有仓库）

**在哪里执行：** SSH 登录到云服务器后，在服务器上的项目根目录执行。

```bash
cd /path/to/your/project   # 替换为实际路径
git pull origin main       # 或你的分支名
```

若后端用虚拟环境：
```bash
cd /path/to/your/project/center_code/backend
source venv/bin/activate   # Linux
pip install -r requirements.txt
```

若前端有更新，在服务器上构建：
```bash
cd /path/to/your/project/center_code/frontend
npm install
npm run build
```

然后重启后端服务（示例）：
```bash
sudo systemctl restart your-backend-service
# 或
supervisorctl restart your-backend
```

### 3.2 方式 B：本地上传文件（无 Git 或习惯用 FTP/SFTP）

1. **本地** 在 `D:\project\333` 确保已拉取并构建：
   - `git pull`
   - `center_code/frontend`: `npm run build`
   - `center_code/backend`: `pip install -r requirements.txt`（可选，看是否改依赖）
2. 用 **FTP/SFTP/SCP** 将以下目录同步到服务器对应位置：
   - `center_code/backend/`（包含 `app.py`、`api/`、`static/`、`requirements.txt`、`.env` 等，注意 `.env` 在服务器上要单独配置或覆盖）。
3. 在 **服务器** 上安装依赖并重启应用（同上）。

## 五、常用命令速查

| 拉取代码 | `D:\project\333` | `git pull` |
