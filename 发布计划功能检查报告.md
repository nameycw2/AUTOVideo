# 发布计划功能检查报告

## 功能概述

发布计划功能允许用户创建定时发布计划，选择平台、账号、视频和发布时间，系统会在指定时间自动执行发布。

## 当前实现状态

### ✅ 已实现的功能

1. **创建发布计划**
   - 前端可以创建发布计划，选择平台、商家、账号、分发模式、发布时间
   - 支持批量发布和分阶段发布两种模式
   - 可以从视频成品库（COS）选择视频

2. **视频管理**
   - 可以向发布计划添加视频（`add_video_to_plan` API）
   - 视频存储在 `PlanVideo` 表中，状态包括：pending、processing、published、failed

3. **定时检查机制**
   - `TaskProcessor` 每分钟检查一次到期的发布计划
   - 当 `publish_time <= now` 时，自动触发处理

4. **任务执行**
   - 处理发布计划时，会为每个视频创建 `VideoTask`
   - 视频任务会被 `execute_video_upload` 执行
   - 支持多平台：douyin、kuaishou、xiaohongshu、weixin、tiktok

5. **状态更新**
   - 视频上传完成后，自动更新 `PlanVideo` 状态为 `published`
   - 自动更新发布计划的统计信息（published_count、pending_count等）
   - 所有视频处理完成后，计划状态更新为 `completed`

### ⚠️ 存在的问题

1. **账号选择未生效**
   - **问题**：前端可以选择账号（`account_ids`），但后端处理时没有使用这些账号
   - **当前行为**：`task_processor.py` 第 230-235 行获取的是"该平台下的所有已登录账号"
   - **影响**：无法按照用户指定的账号发布视频
   - **位置**：`backend/services/task_processor.py:230-235`

2. **账号ID未保存**
   - **问题**：`PublishPlan` 模型中没有存储 `account_ids` 的字段
   - **当前行为**：创建发布计划时，前端传递的 `account_ids` 没有被保存到数据库
   - **影响**：无法在后续处理中使用指定的账号
   - **位置**：`backend/api/publish_plans.py:166-203`

3. **分阶段发布未完全实现**
   - **问题**：分阶段发布模式中，每个视频的 `schedule_time` 被保存到 `publish_schedule`，但后端处理时没有使用
   - **当前行为**：所有视频都使用计划的 `publish_time`，而不是各自的 `schedule_time`
   - **影响**：无法实现真正的分阶段发布（按不同时间发布不同视频）
   - **位置**：`backend/services/task_processor.py:218-221`（只处理 `status='pending'` 的视频，没有考虑各自的发布时间）

4. **分发模式未实现**
   - **问题**：分发模式（manual/sms/qrcode/ai）只是存储，没有实际逻辑
   - **当前行为**：所有模式都使用相同的处理逻辑（轮询分配账号）
   - **影响**：无法实现短信分发、二维码分发、AI智能分发等功能
   - **位置**：`backend/services/task_processor.py:276`（简单的轮询分配）

## 建议修复方案

### 1. 修复账号选择功能

**方案A：在 PlanVideo 中存储账号ID**
- 在 `PlanVideo` 表中添加 `account_id` 字段
- 添加视频时，根据轮询策略分配账号
- 处理时直接使用 `PlanVideo.account_id`

**方案B：在 PublishPlan 中存储账号ID列表**
- 在 `PublishPlan` 表中添加 `account_ids` JSON字段
- 创建/更新计划时保存账号ID列表
- 处理时从 `account_ids` 中获取账号，而不是查询所有账号

**推荐方案B**，因为：
- 不需要修改 `PlanVideo` 表结构
- 更灵活，可以随时修改计划使用的账号
- 符合当前的设计（计划级别指定账号）

### 2. 实现分阶段发布

- 在 `PlanVideo` 表中添加 `schedule_time` 字段
- 添加视频时，保存各自的发布时间
- 处理时，只处理 `schedule_time <= now` 的视频

### 3. 实现分发模式

- **manual**：使用指定的账号列表，轮询分配
- **sms**：发送短信，等待账号确认后分配
- **qrcode**：生成二维码，账号扫码后分配
- **ai**：根据账号历史数据、视频内容等智能分配


```sql
ALTER TABLE `publish_plans` 
ADD COLUMN `account_ids` TEXT NULL COMMENT 'JSON字符串，存储指定的账号ID列表'

ALTER TABLE `plan_videos` 
ADD COLUMN `schedule_time` DATETIME NULL COMMENT '该视频的发布时间（用于分阶段发布）' 
AFTER `thumbnail_url`;
```
