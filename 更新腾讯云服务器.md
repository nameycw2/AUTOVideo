# 腾讯云服务器项目更新说明

服务器地址：`http://1.12.48.189:8081`（已有一版在跑，本次为更新）

---

## 一、在本地先准备好要上传的代码

1. 在本地确认修改已提交（可选）：
   ```bash
   git status
   git add -A
   git commit -m "更新到最新版"
   ```

2. 若使用 Git 部署：推送到远程仓库（服务器上会 `git pull`）。
3. 若不使用 Git：用下面「方式二」直接把本机目录同步到服务器。

---

## 二、在服务器上更新（任选一种方式）

### 方式一：服务器上 Git 拉取（推荐）

SSH 登录服务器后：

```bash
# 1. 进入项目目录（按你实际路径改，常见：/var/www/autovideo_new 或 /var/www/autovideo/AUTOVideo）
export PROJECT_DIR=/var/www/autovideo_new   # 改成你的实际路径
cd "$PROJECT_DIR"

# 2. 拉取最新代码
git pull origin main
# 若主分支是 master：git pull origin master

# 3. 后端依赖（有新增包时）
cd center_code/backend
pip install -r requirements.txt -q
# 若用虚拟环境：source venv/bin/activate 后再 pip install

# 4. 前端依赖并构建（会输出到 center_code/backend/static）
cd "$PROJECT_DIR/center_code/frontend"
npm install --silent
npm run build

# 5. 若有数据库迁移脚本，执行一次（例如）
# cd "$PROJECT_DIR/center_code/backend" && python migrate_xxx.py

# 6. 重启后端
# 若用 systemd：
sudo systemctl restart autovideo
# 若用 nohup 手动起的进程，先停再起：
# pkill -f "python app.py" 2>/dev/null || true
# cd "$PROJECT_DIR/center_code/backend" && nohup python app.py 8081 > /tmp/backend.log 2>&1 &
```

然后浏览器访问：`http://1.12.48.189:8081` 检查是否为新版。

---

### 方式二：本机用 rsync/scp 同步到服务器（不用 Git 时）

在**本机**（项目根目录，即 `d:\project\333` 或对应 Linux 路径）执行：

```bash
# 将 项目根 同步到服务器（排除 node_modules、venv、.git 等）
rsync -avz --exclude 'node_modules' --exclude 'venv' --exclude '.git' --exclude '__pycache__' \
  ./  root@1.12.48.189:/var/www/autovideo_new/
# 若用非 root 用户：ubuntu@1.12.48.189:/var/www/autovideo_new/
```

然后在**服务器**上执行上面「方式一」里的步骤 3～6（安装依赖、构建前端、重启后端）。

## 三、重要配置检查（更新后若异常可看）

| 项 | 说明 |
|----|------|
| 端口 | 后端需监听 8081，访问 `http://1.12.48.189:8081` 才能打开页面。 |
| .env | `center_code/backend/.env` 里数据库、COS、SECRET_KEY 等不要被覆盖，更新时注意保留。 |
| 前端静态 | 当前 Vite 构建输出到 `center_code/backend/static`；Flask 若配置的是 `../frontend/dist`，需保证构建结果被放到 Flask 配置的目录，或把 Flask 改为使用 `static` 目录。 |
| CORS | 若用 IP 访问，确认 `.env` 或 `config` 里 CORS 包含 `http://1.12.48.189:8081`。 |

## 四、一键更新脚本（可选）

项目根目录有 `update.sh`，在服务器上可这样用：

```bash
# 设置项目路径并执行（路径按实际改）
export PROJECT_DIR=/var/www/autovideo_new
bash /var/www/autovideo_new/update.sh
```

执行后仍需要**手动重启后端**（脚本里重启那几行被注释掉了，按你实际用 systemd 或 nohup 自己执行一次）。

## 五、常见问题

- **页面还是旧的**：多半是前端没重新构建或静态目录不对。确保在 `center_code/frontend` 执行了 `npm run build`，且后端服务指向的静态目录就是构建输出目录。
- **接口 404 / 500**：看后端日志（`sudo journalctl -u autovideo -n 100` 或 `/tmp/backend.log`），检查 .env、数据库、依赖是否完整。
- **端口不是 8081**：若你改用 Nginx 反代，确保 Nginx 里 `proxy_pass` 指向实际后端端口（如 8081）。

更新完成后，用 `http://1.12.48.189:8081` 访问即可验证。
